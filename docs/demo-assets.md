# Fictional demonstration assets

Daymark Assembly is the repository's coherent fictional artist. It exists so a judge or new artist can exercise every enabled module without receiving Michael's music, Sound for Movement branding, customer data, production records, or service credentials.

The machine-readable inventory is [`content/demo/assets.json`](../content/demo/assets.json). Every entry is original to this repository, marks `privateMaterial` as false, and names its source and path. The demonstration contains:

- Original fictional identity, catalog, commerce, licensing, learning, video, editorial, and privacy copy.
- Three deterministic one-second PCM tones generated from named frequencies during local seed.
- Trusted deterministic SVG release and lesson artwork generated during local seed.
- One tracked trusted SVG video poster.
- Plain-text learning and protected-download fixtures generated by repository code.
- README screenshots captured from the local fictional installation.

The external video example remains poster-first. No third-party player receives a request until the visitor explicitly consents, and the video itself is not copied into this repository.

## License and publication gate

All demonstration material is original, but it remains `UNLICENSED` while the project license decision is pending. Michael must select the project and asset license and explicitly approve repository publication before any public release. The asset inventory is updated with that decision before the Milestone 12 package is marked complete.

## Replacement contract

An adopting artist replaces Daymark Assembly through the interview, proposal, preview, approval, apply, and verification lifecycle in [`SETUP.md`](../SETUP.md). Rights approval is required for every imported audio, image, video, text, and downloadable file. Generated preview and artwork derivatives retain a hash-addressed relationship to the approved source; the source object is never overwritten.

# Publish learning, video, and editorial work

The learning system preserves the artist's authored order from area to path, course, lesson, and section. Each lesson resolves access through the same authority used by commerce, memberships, licensing, and protected downloads. Video and editorial work use separate private drafts with explicit publication.

## Build and publish a learning path

1. Sign in as an owner or editor and open `/admin/learning`.
2. Create a path or select an existing private draft. Define the learning area, path introduction, courses, lessons, and exact order.
3. Choose an access mode for every lesson:
   - **Public** needs no account.
   - **Account** needs a signed-in customer account.
   - **Individual entitlement** creates an application-owned learning product and one-time price when published. Verified fulfillment grants access to that lesson.
   - **Membership** uses the selected published membership tier and the central membership decision.
4. Compose each lesson from ordered prose, image, audio, video, protected download, and practice-prompt sections.
5. Open the proposed learner sequence. It renders the complete draft structure, safe rich text, private image and audio previews, published video references, resource metadata, and prompts without changing the public path.
6. Save the private draft. Review the confirmation dialog, then publish the exact approved order.

Publishing is transactional. Existing course and lesson identifiers remain stable during reordering, so progress and media ownership survive swaps and moves. Removed units are archived rather than destructively deleting learner history. Individually entitled lessons receive application-owned products and prices; changing their access mode archives the obsolete offering.

## Write safe rich text

Lesson prose accepts a deliberate subset of Markdown:

    **strong emphasis**
    _emphasis_
    [internal label](/learn)
    [approved source](https://example.com)
    - an unordered list item

The public renderer creates Vue text, emphasis, link, and list nodes directly. It never uses raw HTML or `v-html`. Server validation rejects HTML tags, scripts, `javascript:` and HTML data URLs, protocol-relative links, plain HTTP links, and other link targets outside internal paths or HTTPS.

Editorial work at `/admin/editorial` uses the shared structured-section renderer for essays, announcements, learning notes, and information pages. It accepts ordered typed sections rather than pasted HTML. A saved editorial draft remains private until the artist explicitly publishes it.

## Upload and protect lesson media

Save the learning draft before uploading media. The lesson editor hashes the file, requests a signed resumable TUS target, uploads directly to private storage, validates the resulting file signature, and attaches the ready media object by stable identifier.

Supported lesson files are WebP, PNG, JPEG, MP3, WAV, MP4, WebM, PDF, and plain text up to 250 MiB. Uploaded SVG is excluded; the fictional demonstration's SVG was generated by trusted repository code during the seed.

Public lesson media uses the lesson access decision before returning a 60-second signed URL. The artist preview has its own owner/editor-only signed route. Download resources never expose a permanent object URL. A customer without the required membership or entitlement receives HTTP 403, and another customer's progress does not affect the current account.

## Publish accessible video

Open `/admin/videos` to create a private video draft. Choose:

- **YouTube** or **Vimeo** with an approved provider identifier. The public page shows the artist's poster, context, complete transcript, and credits first. The external iframe does not exist until the visitor chooses **Load external video**.
- **Artist-hosted video** using a ready private MP4 or WebM uploaded through a lesson. The public endpoint signs the media after publication and the native video element uses the optional poster.

The demonstration uses video ID `M7lc1UVf-VE`, the sample published in Google's official [YouTube IFrame Player API reference](https://developers.google.com/youtube/iframe_api_reference). It is embedded from YouTube and is not copied into this repository. Replace the sample, transcript, credit, and fictional repository-owned poster with media the artist has approved before a real publication.

## Learner progress and resume

An allowed signed-in learner can save section position or mark a lesson complete. Progress is monotonic: a later request cannot move the saved position backward or undo completion. `/account` derives the next meaningful lesson from published authored order and the current account's progress. Anonymous public lessons remain readable, but anonymous progress is not stored.

## Verify and recover

With local Supabase running, use:

    npm run verify:learning
    npm run seed:reset
    npm run test:e2e -- tests/e2e/learning.spec.ts

The authority test proves order, transactional reordering, access modes, monotonic progress, customer isolation, and individual learning products. The browser suite proves mixed media, protected delivery, membership continuation, account resume, private drafts, server-side injection denial, explicit editorial publication and restoration, privacy-gated external video, safe rich text, complete artist previews, responsive layout, and critical/serious axe checks on public and authenticated surfaces.

If publication fails, keep the draft private and review every referenced media object, video, membership tier, access explanation, price, and section. If a lesson resource is unavailable, confirm it is `ready` in the artist preview and upload it again only when the content hash differs. Do not edit progress, entitlements, or published normalized rows directly; correct the draft or fulfillment source and republish or replay through the supported authority.
